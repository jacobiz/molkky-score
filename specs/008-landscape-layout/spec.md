# Feature Specification: 横画面レイアウト最適化（タブレット対応）

**Feature Branch**: `008-landscape-layout`
**Created**: 2026-03-01
**Status**: Draft
**Input**: User description: "横画面(主にタブレット)での表示を洗練させたい。ポイント入力領域が縦長なので、もう少し横に広げたい(最大で画面の中央まで)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 横画面でポイント入力エリアを広く使えるようにする (Priority: P1)

タブレットを横向きにしてゲームを進行中、ポイント入力エリア（0〜12のボタン）が画面の
右側に現在より広く表示される。現在は右端に細長く固定表示されているが、画面幅に応じて
最大で画面の中央まで拡大することで、ボタンが大きくタップしやすい状態になる。
スコアボードは左側に残り、両エリアが画面を効率的に活用できる。

**Why this priority**: タブレット横画面はゲーム中の主要なユースケースであり、
ポイント入力が操作の中心。ボタンが大きくなると操作ミスが減り、ゲーム体験に直結する。

**Independent Test**: タブレットまたはブラウザの横画面（幅1024px以上）でゲームを開始し、
ポイント入力エリアが以前より広く表示され、かつ画面の半分を超えないことを目視確認する。
各ボタンをタップして得点入力が正常に機能することを確認する。

**Acceptance Scenarios**:

1. **Given** タブレット横画面（幅が十分広い）でゲームが進行中、**When** ゲーム画面を表示する、**Then** ポイント入力エリアが以前の固定幅より広く表示される
2. **Given** ポイント入力エリアが拡大された状態で、**When** 画面幅が非常に広い場合（例: 大型モニター）、**Then** ポイント入力エリアは画面の中央（幅の50%）を超えて拡大しない
3. **Given** ポイント入力エリアが拡大された状態で、**When** ボタンをタップして得点を入力する、**Then** 得点入力が正常に機能し、スコアが更新される
4. **Given** スマートフォンの縦画面でゲームが進行中、**When** ゲーム画面を表示する、**Then** 縦画面のレイアウトは変更されず、従来通り表示される

---

### Edge Cases

- 横画面でも幅が比較的狭い端末（小型タブレット横向き、幅768〜900px程度）で適切に表示されること
- 横画面から縦画面に回転した際に、縦画面レイアウトへ正常に切り替わること
- 画面幅が非常に広いデスクトップ（1920px以上）でも50%上限が機能すること
- スコアボードのプレイヤー一覧が横幅縮小により読みにくくならないこと

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 横画面表示時、ポイント入力エリアの幅が現在より広く表示されること
- **FR-002**: 横画面表示時、ポイント入力エリアの幅は画面幅の最大50%を超えないこと
- **FR-003**: 横画面表示時、スコアボードエリアはポイント入力エリアの拡大に応じて残余スペースを使用すること
- **FR-004**: 縦画面表示時のレイアウトは本機能による変更を受けないこと
- **FR-005**: 既存のゲーム機能（スコア入力・undo・ターン管理）が横画面拡大後も正常に動作すること

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 横画面表示時（1024px以上）、ポイント入力エリアの幅が変更前より広くなっていること（目視確認）
- **SC-002**: 横画面表示時、ポイント入力エリアが画面幅の50%を超えないこと（目視確認）
- **SC-003**: 縦画面表示時、既存レイアウトに変化がないこと（目視確認）
- **SC-004**: 横画面でのポイント入力・undo・スコア表示が正常に動作すること
- **SC-005**: 既存の全ユニットテストが引き続きパスすること

## Assumptions

- 「横画面」とは現在の横並びレイアウトが適用される画面幅（768px以上）を指す
- ポイント入力エリアは画面幅に応じて拡大し、上限は画面幅の50%とする
- 有効な幅改善が確認できる主対象は1024px以上の横画面（タブレット・デスクトップ）とする。768〜800px 横画面では改善幅が微小または現状維持となる場合がある
- 縦画面（768px未満）のレイアウトは本機能の対象外
- Mölkkout 画面は本機能のスコープ外とする
