# Feature Specification: Back to Home Navigation

**Feature Branch**: `011-game-back-home`
**Created**: 2026-03-01
**Status**: Draft
**Input**: User description: "TOP以外のすべての画面からホーム画面に戻れるようにしたい。確認ダイアログつき"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Exit to Home from Game Screen (Priority: P1)

ゲームプレイ中にプレイヤーがゲームを中断してホーム画面に戻りたい場合、ゲーム画面に配置された「ホームへ戻る」ボタンをタップすると確認ダイアログが表示され、確認後にホーム画面へ遷移できる。ゲームデータは保持されるため「ゲームを再開」で続行できる。

**Why this priority**: 最も頻度が高いユースケースであり、このストーリー単独で機能として成立する。

**Independent Test**: ゲーム画面でホームボタンをタップ → 確認ダイアログ表示 → 確認 → ホーム画面遷移、の流れを確認するだけでテスト可能。

**Acceptance Scenarios**:

1. **Given** ゲームが進行中の状態で, **When** プレイヤーがホームへ戻るボタンをタップする, **Then** 「ホームへ戻りますか？」の確認ダイアログが表示される
2. **Given** 確認ダイアログが表示されている状態で, **When** プレイヤーが「確認（はい）」を選択する, **Then** ホーム画面へ遷移する
3. **Given** 確認ダイアログが表示されている状態で, **When** プレイヤーが「キャンセル」を選択する, **Then** ダイアログが閉じてゲーム画面に戻り、ゲーム状態は変化していない
4. **Given** ホームへ戻った後, **When** 「ゲームを再開」を選択する, **Then** 中断時点のゲーム状態が完全に復元されている

---

### User Story 2 - Exit to Home from Setup Screen (Priority: P2)

プレイヤー設定画面（ゲーム開始前）でホームへ戻るボタンをタップすると、確認ダイアログが表示され、確認後にホーム画面へ遷移できる。入力中のプレイヤー情報は破棄される。

**Why this priority**: セットアップ画面に入ったユーザーがホームに戻る手段が現状存在しない。P1 の確認ダイアログが実装済みであれば追加コストは低い。

**Independent Test**: セットアップ画面でプレイヤー名を入力してからホームボタンをタップし、確認後にホーム画面へ遷移することを確認するだけでテスト可能。

**Acceptance Scenarios**:

1. **Given** プレイヤー設定画面にいる状態で, **When** ホームへ戻るボタンをタップする, **Then** 確認ダイアログが表示される
2. **Given** 確認を選択した場合, **Then** ホーム画面へ遷移し、入力済みのプレイヤー情報は保持されない

---

### User Story 3 - Exit to Home from Result / Mölkkout Screens (Priority: P3)

結果画面・Mölkkout（タイブレーカー）画面でも同一のホームへ戻る操作が利用できる。

**Why this priority**: 一貫したナビゲーション体験のため。これらの画面にはすでに一部のナビゲーションが存在するが、統一されていない。

**Independent Test**: 結果画面でホームボタンをタップしてホーム画面へ到達できることを確認するだけでテスト可能。

**Acceptance Scenarios**:

1. **Given** 結果画面にいる状態で, **When** ホームへ戻るボタンをタップする, **Then** ホーム画面へ遷移する（ゲームが終了済みのため確認ダイアログは不要）
2. **Given** Mölkkoutゲーム中にいる状態で, **When** ホームへ戻るボタンをタップする, **Then** 確認ダイアログが表示され、確認後にホーム画面へ遷移する

---

### Edge Cases

- デバイスのOSレベルの「戻る」ジェスチャー（Android）の挙動：本機能の対象外とし、アプリ内ボタン操作のみを対象とする
- 結果画面のホームボタン：ゲームが既に終了しているため確認ダイアログは不要
- LicensesScreen（ライセンス表示）：ホーム画面内のオーバーレイとして動作しているため対象外

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: TOP（ホーム）画面を除くすべての画面（setup・game・result・molkkout-setup・molkkout-game）に「ホームへ戻る」ボタンが表示されていなければならない
- **FR-002**: 確認ダイアログを表示する画面は **setup・game・molkkout-game の3画面** に限定する。result・molkkout-setup はゲーム終了後の遷移画面のため確認ダイアログは不要とする
- **FR-003**: 確認ダイアログには「ホームへ戻る（確認）」と「キャンセル」の2つの選択肢が必ず含まれなければならない
- **FR-004**: 確認を選択した場合、ホーム画面へ遷移しなければならない
- **FR-005**: キャンセルを選択した場合、ダイアログを閉じて元の画面を現在の状態のまま維持しなければならない
- **FR-006**: ゲームプレイ中にホームへ戻った後も、ゲームデータは保持され「ゲームを再開」で復帰できること
- **FR-007**: ホームへ戻るボタンは**共通ヘッダ**として実装し、対象全5画面の最上部に統一されたヘッダバーを配置しなければならない。ヘッダにはホームへ戻るボタンと現在の画面タイトルを含むこと
- **FR-008**: ホームへ戻るボタンの表示文言は既存の多言語対応（日本語・英語・フィンランド語）に従うこと

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: TOP画面を除くすべての画面で、プレイヤーが2回以内のタップ操作でホーム画面へ到達できる
- **SC-002**: 進行中データがある画面でホームへ戻るボタンを押した場合、100%の操作に対して確認ダイアログが表示される
- **SC-003**: キャンセル後に画面を再確認したとき、スコア・ターン数・プレイヤー順序がキャンセル前と完全に一致している
- **SC-004**: ゲームデータのあるホーム遷移後に「ゲームを再開」を選択した場合、中断ポイントのゲーム状態が100%復元される
- **SC-005**: すべての対象画面（5画面）でホームへ戻るボタンが同一の操作体験で利用できる

## Clarifications

### Session 2026-03-01

- Q: molkkout-setup 画面でホームへ戻る際に確認ダイアログは必要か？ → A: 不要（result 画面と同じ扱い。ゲーム終了後の遷移画面のため失われるデータなし）

## Assumptions

- 対象画面: setup / game / result / molkkout-setup / molkkout-game の5画面（LicensesScreen はホーム画面内オーバーレイのため対象外）
- 確認ダイアログが不要な画面: result・molkkout-setup（ゲーム終了後の遷移画面のため、失われるデータなし）
- 確認ダイアログが必要な画面: setup・game・molkkout-game（進行中データまたは入力中データが存在する）
- ゲームデータはすでに既存の仕組みで永続化されているため、データ保持は追加実装不要
- ホームへ戻るボタンは共通ヘッダとして全対象画面に配置する（デザイン詳細はプランニングフェーズで確定）
