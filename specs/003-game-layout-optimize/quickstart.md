# Quickstart: ゲーム画面レイアウト最適化 動作確認

**Feature**: 003-game-layout-optimize
**前提**: `npm run dev` でアプリを起動し、スマートフォンまたはブラウザのモバイルエミュレーター（DevTools → iPhone SE / iPhone 14 等）で確認

---

## シナリオ 1: 2人ゲーム - ボタンが大きく表示される

1. ホーム → 「新しいゲーム」
2. プレイヤーを 2人追加（例: A, B）
3. ゲーム開始
4. **確認**: スコア入力エリア（0〜12のボタン）が画面下半分以上を占めており、ボタンが大きい
5. **確認**: 0（赤）ボタンと 1〜12（緑）グリッドがすべて画面内に収まっている

---

## シナリオ 2: 6人ゲーム - ボタンが画面内に収まる

1. ホーム → 「新しいゲーム」
2. プレイヤーを 6人追加（例: A, B, C, D, E, F）
3. ゲーム開始
4. **確認**: スコア入力ボタン（0〜12）がスクロールなしで全ボタン表示・操作できる
5. **確認**: スコアボードが 5〜6行程度表示されており、下の方の行は少しスクロールで見える
6. **確認**: 各ボタンの高さが十分あり、誤タップしにくいサイズである（≥44px）

---

## シナリオ 3: 6人ゲーム - 脱落時レイアウト安定

1. 6人でゲーム開始
2. 1人目のプレイヤーで 0 を 3回入力（脱落）
3. **確認**: 脱落後もスコア入力ボタンのサイズ・位置が変わらない（レイアウトシフトなし）
4. 次のプレイヤーでスコアを入力できることを確認

---

## シナリオ 4: 2人 → 6人の比較

1. 2人でゲームを開始し、ボタンの高さを目視で確認
2. 戻ってリスタートし、6人でゲームを開始
3. **確認**: 6人時は 2人時よりボタンが小さくなるが、それでも十分タップできるサイズである

---

## シナリオ 5: Mölkkout - ボタンが画面内に収まる

1. ホーム → Mölkkout（タイブレーカー）セットアップ
2. 2〜3チームを設定してゲーム開始
3. **確認**: Mölkkout のスコア入力ボタン（0〜12）がスクロールなしで全表示・操作できる
4. **確認**: チームスコアリストとボタンが適切に分割されている

---

## シナリオ 6: タブレット / デスクトップ - 既存レイアウト維持

1. ブラウザを幅 768px 以上にリサイズ（または iPad エミュレーション）
2. 4〜6人でゲーム開始
3. **確認**: 左ペイン（スコアボード）と右ペイン（入力）が横並びになっている
4. **確認**: 機能変更前と見た目が変わらない

---
