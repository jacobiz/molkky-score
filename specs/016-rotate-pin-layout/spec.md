# Feature Specification: Mölkkout ピン配置図の縦向き表示

**Feature Branch**: `016-rotate-pin-layout`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "前回の修正(015)で追加したモルックアウト時のピン配置図を、現在の横向きから縦向き(スロワー視点で正しい向きでの表示になるよう)に回転させたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Mölkkout ピン配置を縦向きで確認する (Priority: P1)

Mölkkout ゲームをセットアップしようとしているプレイヤーが、配置ガイドを開いたとき、実際にフィールドでピンを置くときの視点（スロワーが前に立ち、ピンが遠ざかる方向に並ぶ）と同じ向きで配置図を見たい。現在の横向き（左右並び）表示では、実際のフィールドの向きと一致せず、どのピンが「前列」でどれが「後列」かが直感的にわからない。

**Why this priority**: 配置ガイドの存在意義はピンを正しく素早く並べることにある。向きが実感とずれていると、ガイドを見ながら配置する際に混乱が生じる。この修正が唯一のユーザーストーリーであり、最優先。

**Independent Test**: Mölkkoutゲーム画面から配置ガイドを開き、ピン配置図が縦向き（スロワー側を下、遠方ピンを上）に表示され、投球ラインの方向インジケーターが下側（スロワー側）に表示されることを確認することで単独テストできる。

**Acceptance Scenarios**:

1. **Given** Mölkkoutゲーム画面の配置ガイドモーダルが開いている状態で、**When** プレイヤーが配置図を確認する、**Then** 5本のピン（#6・#4・#12・#10・#8）が縦一列に表示され、最前列ピン（#6）が最も下（スロワー側）、最後列ピン（#8）が最も上に配置されている
2. **Given** Mölkkoutゲーム画面の配置ガイドモーダルが開いている状態で、**When** プレイヤーが配置図を確認する、**Then** スロワーの方向（下）を示す視覚的インジケーター（投球ラインや矢印など）が配置図の下部に表示される
3. **Given** Mölkkoutゲーム画面の配置ガイドモーダルが開いている状態で、**When** プレイヤーが配置図を確認する、**Then** ピン間隔（1 Mölkky スティック分）の情報が引き続き表示される
4. **Given** 配置ガイドモーダルが開いている状態で、**When** プレイヤーが閉じる操作を行う、**Then** 通常通りモーダルが閉じてゲーム画面に戻る（既存動作に変化なし）

---

### Edge Cases

- 縦向き表示にしたことで、モーダルの縦方向のスペースが不足した場合はどうなるか？ → モーダルは既に `overflow-y-auto` でスクロール可能なため、表示が崩れることなくスクロールで全体を確認できる
- 通常ゲーム（4行三角配置）の配置図には変更を加えないため、通常ゲームの表示に影響しないことを確認する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Mölkkoutの配置ガイドは、5本のピン（#6・#4・#12・#10・#8）を縦一列に表示しなければならない（MUST）。表示順は上から #8・#10・#12・#4・#6（スロワー視点で奥から手前の順）
- **FR-002**: 縦向き配置図の下部（スロワー側）に、スロワーの方向・投球ラインを示すビジュアルインジケーターを表示しなければならない（MUST）
- **FR-003**: ピン間隔（1 Mölkky スティック分）の情報表示を引き続き含まなければならない（MUST）
- **FR-004**: 通常ゲームの配置図（4行三角配置）は変更してはならない（MUST NOT）
- **FR-005**: 既存の全テキスト（距離ラベル・間隔ラベル・リセット注記）は i18n 対応を維持しなければならない（MUST）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Mölkkout配置ガイドを見たプレイヤーが、どのピンが「スロワーに最も近い前列」かを一目で識別できる（縦向き表示で直感的に理解できる）
- **SC-002**: 配置図の向きを縦に変更した後も、配置ガイドモーダルが正しく開閉でき、ゲーム状態への副作用がゼロである
- **SC-003**: 通常ゲーム（4行三角配置）の配置ガイドの表示・動作が変更前と100%同一である

## Assumptions *(optional)*

- **A-001**: Mölkkout の5ピン配置における「前列から後列への順序」は #6（最前列）→ #4 → #12 → #10 → #8（最後列）とする（015 スペックの定義に従う）
- **A-002**: 縦向き表示では、ピン同士の接続を示す区切り記号（現行の「—」）は縦方向に変更するか省略する。視覚的な一列感が伝われば記号の形式は問わない
- **A-003**: 通常ゲームの配置図（RegularLayout）は変更対象外。スコープはMölkkout配置図（MolkkoutLayout）のみ
