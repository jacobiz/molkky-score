# Feature Specification: Shuffle Player Order

**Feature Branch**: `005-shuffle-order`
**Created**: 2026-03-01
**Status**: Draft
**Input**: User description: "ゲーム開始前のプレイヤー入力後に、投球順をランダマイズする機能を追加したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 投球順をシャッフルする (Priority: P1)

プレイヤー登録画面で全員を入力し終えたあと、「シャッフル」ボタンを1回タップするだけで投球順をランダムに並び替えられる。毎回手動で並び替える手間がなくなり、公平なゲーム開始が即座に実現できる。

**Why this priority**: ゲームの公平性を高める最もシンプルかつ直接的な操作。実装も独立していてリスクが低く、これだけで機能として完結する。

**Independent Test**: プレイヤーを3人以上登録し「シャッフル」を複数回タップして順番が変わることを確認できる。

**Acceptance Scenarios**:

1. **Given** 2人以上のプレイヤーが登録済み、**When** シャッフルボタンをタップ、**Then** プレイヤーリストの順番がランダムに並び替えられる
2. **Given** シャッフル後の順番が表示されている、**When** さらにシャッフルボタンをタップ、**Then** 再度ランダムに並び替えられる（偶然同じ順になる場合もある）
3. **Given** シャッフル後の順番が表示されている、**When** ゲームを開始、**Then** シャッフル後の順番でゲームが進行する
4. **Given** シャッフル後の順番が表示されている、**When** 上へ／下へボタンで手動調整、**Then** 手動調整後の順番が反映される

---

### Edge Cases

- プレイヤーが1人の場合：シャッフルボタンは非表示（並び替えの意味がないため）
- プレイヤーが0人の場合：シャッフルボタンは非表示
- シャッフル後に手動で順番を変更することは引き続き可能
- シャッフルで偶然もとの順番と同じになることは仕様として許容

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: プレイヤーが2人以上登録されている場合に限り、「シャッフル」ボタンを表示する
- **FR-002**: シャッフルボタンをタップすると、登録済みプレイヤーの順番を無作為に並び替える
- **FR-003**: シャッフル後も手動での順番変更（上へ／下へボタン）を引き続き利用できる
- **FR-004**: シャッフル後の順番でゲームを開始した場合、その順番がゲーム内の投球順となる
- **FR-005**: プレイヤーが0人または1人のときはシャッフルボタンを表示しない

### Key Entities

- **Player Order**: プレイヤーリストの並び順。シャッフルにより無作為に更新される。プレイヤー名や個々の設定は変化しない。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: シャッフルボタンのタップから順番更新まで視覚的に即座（100ms以内）に反映される
- **SC-002**: シャッフル後にゲームを開始すると、シャッフル後の順番通りにゲームが進行する（1ターン目のプレイヤーが一致する）
- **SC-003**: シャッフル導入後も既存の手動並び替え機能（上へ／下へ）が正常に動作する

## Assumptions

- シャッフルアルゴリズムは標準的なランダム化（Fisher-Yates 等）を使用する
- シャッフルボタンは既存のプレイヤーリスト操作エリア（上へ／下へボタンと同じ画面）に配置する
- シャッフル確認ダイアログは不要（タップ1回で即座に実行）
- プレイヤー名・設定はシャッフルの影響を受けない（順番のみが変わる）
- i18n（日英対応）は既存の仕組みに従う
