# Feature Specification: Game UI Polish

**Feature Branch**: `004-game-ui-polish`
**Created**: 2026-03-01
**Status**: Draft
**Input**: User description: "UIをよくしたい。・ゲーム中画面のヘッダに、「今何ターン目(何投目)」を表示したい ・文字をほんの少しだけ大きくして視認性を上げたい ・「残り{n}」の{n}が12以下になったら(あと1投で終わる可能性あり)目立つようにしたい ・「やり直し」ボタンに絵文字やアイコンを追加してundoが直感的にわかるようにしたい。他にも絵文字やアイコンを使うことでUI・UXがよくなる点を提案してほしい"

## Clarifications

### Session 2026-03-01

- Q: ヘッダ内の投数とプレイヤー名の配置はどうするか？ → A: 2段表示（上段: プレイヤー名、下段: N投目）。NはゲームI全体の累計ではなく、各プレイヤーの個別投数。
- Q: 投数表示のi18n対応は必要か？ → A: 必要。言語設定に従い切り替える（ja: "N投目" / en: "Nth throw"）。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ターン数ヘッダ表示 (Priority: P1)

プレイヤーがゲーム中画面を見たとき、ヘッダ上段に「{name} のターン」、下段に「N投目（現在のプレイヤーがこのゲームで何投目か）」が表示されており、誰がいま投げるのか・そのプレイヤーが今まで何回投げたかを瞬時に把握できる。

**Why this priority**: 各プレイヤーの投球回数は戦略的判断（ミス数と合わせてリスク判断など）や振り返りに役立つ。2段レイアウトにより既存のプレイヤー名表示を損なわず追加できる。

**Independent Test**: ゲームを開始し、同一プレイヤーが複数回投球するまでゲームを進める。そのプレイヤーのターンになるたびに下段の数値が1ずつ増え、他のプレイヤーのターンでは変わらないことで独立して検証できる。

**Acceptance Scenarios**:

1. **Given** ゲームを新たに開始した、**When** 最初のプレイヤーのターンになった、**Then** ヘッダ上段に「{name} のターン」、下段に「1投目」が表示される
2. **Given** プレイヤーAが1投目を終えてプレイヤーBのターンになった、**When** ヘッダを確認する、**Then** 上段に「{B} のターン」、下段に「1投目」（Bの初投）が表示される
3. **Given** プレイヤーAが2巡目（全プレイヤーが1回ずつ投げ終えた後）のターンになった、**When** ヘッダを確認する、**Then** 下段に「2投目」（Aの2回目）が表示される
4. **Given** ゲームが進行中、**When** 「やり直し」を実行する、**Then** 現在のプレイヤーの投数が1つ戻る（直前に増えた分が取り消される）

---

### User Story 2 - 残りスコア警告表示 (Priority: P2)

あるプレイヤーの残りスコア（50点到達に必要な残り点数）が12点以下になったとき、そのプレイヤーの残り表示が通常と異なる目立つ見た目になる。これにより、そのプレイヤーが次の1投で優勝する可能性があることを全員がすぐに気づける。

**Why this priority**: 残り12点以下とは「次の1投で50点ちょうどに到達しうる」状態であり、ゲームの緊張感が最高潮に達するタイミング。この情報を視覚的に際立たせることで、観戦している全プレイヤーのエンゲージメントが高まる。

**Independent Test**: プレイヤーのスコアを操作して残り12点・13点の境界前後で表示の変化を確認する。独立して検証可能。

**Acceptance Scenarios**:

1. **Given** プレイヤーAのスコアが38点（残り12点）、**When** スコアボードを表示する、**Then** プレイヤーAの残りスコア表示が目立つ強調スタイルになっている
2. **Given** プレイヤーAのスコアが37点（残り13点）、**When** スコアボードを表示する、**Then** プレイヤーAの残りスコアは通常スタイルで表示される
3. **Given** プレイヤーAの残り12点（強調表示中）、**When** プレイヤーAがバースト（50点超えで25点リセット）する、**Then** プレイヤーAの残り表示は通常スタイルに戻る（残り25点）
4. **Given** プレイヤーAの残り0点（50点ちょうどで勝利）、**When** 勝利判定が出た、**Then** 通常の勝利フローが動作し残り強調は結果に影響しない
5. **Given** 脱落したプレイヤーのスコアが38点だった場合、**When** スコアボードを表示する、**Then** 脱落ラベルが優先表示され、残りの強調は表示しない

---

### User Story 3 - スコアボード文字サイズ改善 (Priority: P3)

スコアボードの文字サイズを現状より若干大きくし、屋外の明るい環境や素早い視線移動でも各プレイヤーのスコアを読み取りやすくする。

**Why this priority**: 視認性改善は体験向上として価値があるが、P1・P2に比べて直接的なゲームプレイへの影響は軽微なため優先度を下げる。

**Independent Test**: スコアボードを表示して各数値・ラベルのフォントサイズが変更前より大きくなっていることを確認できる。

**Acceptance Scenarios**:

1. **Given** ゲームが進行中、**When** スコアボードを表示する、**Then** スコア・残り・ミス数などの数値が現状より1段階大きいサイズで表示される
2. **Given** 6人フルゲーム（プレイヤー最大数）、**When** スコアボードを表示する、**Then** 文字サイズ拡大後もスコアボードが画面内に収まり、スクロール不要な範囲は損なわれない

---

### User Story 4 - アクションボタンへのアイコン追加 (Priority: P4)

ゲーム・セットアップ・ホーム・結果の各画面で、主要なアクションボタンにアイコンを付与することで、テキストを読む前に操作の意味が直感的に伝わるようにする。特に「やり直し」ボタンはモルック中に素早くタップされるため、undo を表すアイコンで即座に認識できることが重要。

**Why this priority**: アイコンは言語に依存せず操作を伝えられるため、日英両言語ユーザーにとっての直感性が上がる。「やり直し」は誤操作の取り消しという重要操作であり特に優先する。他のボタンへの追加は付帯効果として品質を底上げする。

**Independent Test**: 各ボタンのアイコン有無を比較し、初見ユーザーがテキストを読まずにボタン目的を正しく推測できるかで検証できる。

**Proposals（提案一覧）**:

| 場所 | ボタン / ラベル | 提案アイコン | 優先度 |
|------|----------------|------------|--------|
| ゲーム中画面 | 「やり直し」undo ボタン | ↩️ undo矢印 | 必須（明示要望） |
| ホーム画面 | 「新しいゲームを始める」 | ▶️ 再生 | 推奨 |
| ホーム画面 | 「ゲームを再開」 | ⏸️→▶️ or ⏮️ | 推奨 |
| セットアップ画面 | 「ゲーム開始」 | ▶️ 再生 | 推奨 |
| 結果画面 | 「結果をシェア」 | 📤 共有 | 推奨 |
| 結果画面 | 「もう一度プレイ」 | 🔄 リプレイ | 任意 |

**Acceptance Scenarios**:

1. **Given** ゲーム中画面を表示している、**When** やり直しボタンを見る、**Then** テキスト「やり直し」に加えてundo/戻るを示すアイコンが表示されている
2. **Given** ホーム画面を表示している、**When** 新規ゲームボタンを見る、**Then** テキストに加えて進む/開始を示すアイコンが表示されている
3. **Given** 結果画面を表示している、**When** シェアボタンを見る、**Then** テキストに加えて共有を示すアイコンが表示されている

---

### User Story 5 - ゲーム内状態アイコン (Priority: P5)

スコアボードや通知（トースト）において、状態を表すアイコンを追加することで、テキストを追わなくても各プレイヤーの状況（脱落・残り僅か・優勝）が一目でわかるようにする。複数人が1画面を覗き込む屋外利用では特に効果的。

**Why this priority**: ゲーム状態の視覚的明確さはゲーム体験の質に直結するが、アクションボタンほど即時性は求められないため P5 とする。

**Independent Test**: 各状態アイコンの有無を比較し、スコアボードをざっと見たときに各プレイヤーの状態を正しく把握できるかで検証できる。

**Proposals（提案一覧）**:

| 場所 | 状態 / ラベル | 提案アイコン |
|------|-------------|------------|
| スコアボード | 「脱落」バッジ | ❌ または 🚫 |
| スコアボード | 現在のターンプレイヤー | ▶️ または 🎯（行の先頭） |
| 結果画面 | 優勝者表示 | 🏆 トロフィー |
| バーストトースト | バースト通知 | 💥 爆発 |

**Acceptance Scenarios**:

1. **Given** プレイヤーが脱落している、**When** スコアボードを表示する、**Then** そのプレイヤーの行に脱落を示すアイコンが表示される
2. **Given** ゲームが終了した、**When** 結果画面を表示する、**Then** 優勝者の名前の近くにトロフィー等の勝利を示すアイコンが表示される
3. **Given** プレイヤーがバーストした（50点超え）、**When** バースト通知が表示される、**Then** 通知にバーストを示すアイコンが含まれている

---

### Edge Cases

- 各プレイヤーの初投（そのプレイヤーが今ゲームで初めて投げるとき）は「1投目」が表示される
- やり直し（undo）を各プレイヤーの1投目に対して行った場合、投数が0にはならず最小1を保つ（undo不可の場合も同様）
- 残りスコア12点ちょうどと13点の境界：12以下が強調、13以上は通常
- バースト後（25点リセット）：残りが25点になり、強調は解除される
- 脱落プレイヤー：残りスコアが12以下であっても脱落表示を優先し、残りの強調は表示しない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ゲーム中画面のヘッダを2段にし、上段に「{name} のターン」、下段に現在のプレイヤーの個別投数を常時表示する。投数の表示文言は言語設定に従い切り替わる（日本語: 「N投目」、英語: 「Nth throw」）
- **FR-002**: 各プレイヤーの投数は独立してカウントし、自分のターンが回ってくるたびに1増える（他のプレイヤーが投げても変わらない）
- **FR-003**: やり直し（undo）を実行すると、現在のプレイヤーの投数が1減る（直前の投球が取り消されるため）
- **FR-004**: スコアボードで各プレイヤーの「残り」点数が12点以下の場合、その値を視覚的に強調表示する
- **FR-005**: 残りスコアの強調表示は、残りが13点以上になった時点（バースト等）で通常表示に戻る
- **FR-006**: 脱落したプレイヤーは、残りスコアが12点以下であっても強調表示を適用しない
- **FR-007**: スコアボードの数値・ラベルのフォントサイズを現状より1段階大きくし、視認性を向上させる
- **FR-008**: ゲーム中画面の「やり直し」ボタンにundoを示すアイコンを追加する（テキストと併用）
- **FR-009**: ホーム・セットアップ・結果の各画面の主要アクションボタン（新規ゲーム・ゲーム再開・ゲーム開始・結果シェア・もう一度プレイ）にそれぞれの操作を示すアイコンを追加する（テキストと併用）
- **FR-010**: スコアボードの脱落プレイヤー行に脱落状態を示すアイコンを表示する
- **FR-011**: 結果画面の優勝者表示に勝利を示すアイコン（トロフィー等）を追加する
- **FR-012**: バースト発生時の通知にバーストを示すアイコンを追加する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ゲーム中画面のヘッダを確認することで、現在のプレイヤーが今ゲームで何投目かが1秒以内に把握できる
- **SC-002**: 残りスコアが12点以下のプレイヤーを、スコアボードを一瞥するだけで通常プレイヤーと区別できる（視覚的差異が明確）
- **SC-003**: 文字サイズ改善後、スコアボードの数値が腕を伸ばした状態（画面を遠目に見る）でも読み取れる大きさになる
- **SC-004**: 6人フルゲームでも、スコアボードが画面内に収まる（スクロール不要な範囲が維持される）
- **SC-005**: 「やり直し」ボタンのアイコンを見た初見ユーザーが、テキストを読まずにundoと認識できる
- **SC-006**: 各画面の主要アクションボタンを一瞥したとき、アイコンによりボタンの目的がテキスト補助なしに推測できる

## Assumptions

- 「N投目」のNは、そのプレイヤーがこのゲームで投球した累計回数（他のプレイヤーの投球は含まない）
- 投数表示の英語形式（"Nth throw"）は序数（1st, 2nd, 3rd, 4th...）を使用する。既存の `rankSuffix` と同様のロジックを参考にできる
- 残りスコアの強調は色・太さ等の視覚的変化で実現し、アニメーションは必須としない
- フォントサイズの「ほんの少し大きく」は1段階のサイズアップを意味し、大幅な拡大は想定しない
- 既存のレイアウト制約（スコアボードの高さ上限など、前フィーチャーで設定済みのもの）は維持する
- アイコンはテキストと必ず併用し、アイコン単独に置き換えない（アクセシビリティ・i18n の観点）
- Proposals（提案一覧）表は実装時の参考候補であり、最終的な選択は計画フェーズで確定する
