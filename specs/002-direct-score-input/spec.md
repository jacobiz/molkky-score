# Feature Specification: 得点直接入力（1タップスコア入力）

**Feature Branch**: `002-direct-score-input`
**Created**: 2026-02-28
**Status**: Draft
**Input**: 得点入力を1タップで完了させたい。(本数入力ではなくて、入力点を0-12で直接入力させる)。背景として、プレイヤー側での得点計算の不可が低いため、入力コストを下げたい

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 通常ゲームでの得点直接入力 (Priority: P1)

投球後、プレイヤーは「何本倒れた」「ピン番号は何番」という2ステップの思考を経ずに、すでに頭の中にある「何点取れた」という答えをそのままタップして記録できる。0〜12の得点ボタンが一画面に並び、1タップで入力が完了する。

**Why this priority**: スコア入力はゲーム中に最も頻繁に行う操作であり、入力コストの削減が最も大きなユーザー価値をもたらす。現行の2ステップ入力（本数 → ピン番号）はプレイヤーが自力で計算・番号確認をする必要があり、会話の流れを妨げる。

**Independent Test**: ゲーム画面を開き、得点ボタンを1回タップするだけでスコアが更新され次のプレイヤーに移れば完成。

**Acceptance Scenarios**:

1. **Given** プレイヤーAのターンで累積スコアが30点のとき、**When** 得点ボタン「12」を1回タップする、**Then** スコアが30+12=42点に更新され次のプレイヤーに移る（2ステップ入力は発生しない）
2. **Given** プレイヤーBのターンで累積スコアが20点のとき、**When** 得点ボタン「3」を1回タップする、**Then** スコアが20+3=23点に更新される
3. **Given** プレイヤーCのターンのとき、**When** 得点ボタン「0」を1回タップする（ミス）、**Then** スコアは変わらず連続ミスカウントが1増える
4. **Given** プレイヤーDのスコアが48点のとき、**When** 得点ボタン「5」を1回タップする、**Then** バースト判定が働きスコアが25点にリセットされる

---

### User Story 2 - Mölkkout モードでの得点直接入力 (Priority: P2)

Mölkkout（タイブレーカー）モードでも同じ1タップ入力を使用できる。Mölkkout の5本ピン（6-4-12-10-8）で得られる得点は 0・4・6・8・10・12点のいずれかだが、プレイヤーが得点値を直接タップして入力できる。

**Why this priority**: Mölkkout でも同じ操作体験を提供することで学習コストを0に抑えられる。ただし通常ゲームよりプレイ頻度が低いため P2。

**Independent Test**: Mölkkout 画面で得点ボタンを1回タップするだけでチームスコアが更新されれば完成。

**Acceptance Scenarios**:

1. **Given** チームAの投球者ターンのとき、**When** 得点ボタン「12」を1回タップする、**Then** チームAの合計スコアに12が加算され次の投球者に移る
2. **Given** チームBの投球者ターンのとき、**When** 得点ボタン「0」を1回タップする、**Then** チームBのスコアは変わらず次の投球者に移る

---

### Edge Cases

- 0点ボタン（ミス）を誤タップしないよう、視覚的に他の得点ボタンと区別できるか
- アンドゥ操作は1タップ入力後でも機能し、直前のターンを取り消せること
- バースト（累積 > 50 → 25点リセット）・勝利判定（累積 = 50）は得点直接入力後も自動で動作すること
- 3回連続ミスによる脱落判定は0点入力後も自動で動作すること

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 得点入力画面は 0〜12 の整数ボタン（計13個）を一画面に表示し、1回のタップで入力を完了しなければならない
- **FR-002**: 得点ボタンをタップした瞬間に当該プレイヤーのスコアが更新され、次のプレイヤーへターンが移らなければならない（確認ステップ・ピン番号入力ステップ不要）
- **FR-003**: バースト判定（累積 > 50 → 25点リセット）・脱落判定（3回連続ミス）・勝利判定（累積 = 50）は得点直接入力後も自動で行われなければならない
- **FR-004**: アンドゥ操作は1タップ入力後でも機能し、直前のターンのスコアと状態を完全に復元できなければならない
- **FR-005**: Mölkkout モードの入力も同じ 0〜12 の1タップ方式でなければならない
- **FR-006**: 0点ボタン（ミス）は視覚的に他の得点ボタンと区別できなければならない

### Key Entities

- **得点（Points）**: プレイヤーが1ターンで得る点数。0〜12の整数。バースト・勝利・脱落判定のインプットとなる。

---

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 得点入力が1タップで完了する（入力操作が現行の最大2ステップから1ステップに削減される）
- **SC-002**: 既存のルール判定（バースト・脱落・勝利）・アンドゥ機能がすべて正常に動作し続ける（リグレッションなし）
- **SC-003**: Mölkkout モードでも同じ1タップ入力が動作する

---

## Assumptions

- モルックのルール上、1ターンの得点は 0〜12 の整数に収まる（12本ピンで最大12本倒せるため）
- Mölkkout モードでも 0〜12 の入力範囲で全パターンをカバーできる
- ピン番号の情報（どの番号のピンを倒したか）はスコア計算には不要であり、ターン履歴への記録も廃止する（統計分析機能は現スコープ外）
- 既存のアンドゥ・バースト・勝利・脱落の各ロジックは変更せず、入力インターフェースのみ変更する
