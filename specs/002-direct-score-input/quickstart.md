# Quickstart: 得点直接入力（1タップスコア入力）

**Branch**: `002-direct-score-input` | **Date**: 2026-02-28

---

## 手動テストシナリオ

### シナリオ 1: 基本的な1タップ入力

1. アプリを起動しプレイヤー2人でゲームを開始する
2. ゲーム画面に 0〜12 のボタンが表示されることを確認する
3. ボタン「12」を1回タップする
4. **期待値**: スコアが12点に更新され次のプレイヤーに移る（2ステップ目のピン番号選択が出ない）

### シナリオ 2: ミス入力（0点）

1. ゲーム画面で「0」ボタンをタップする
2. **期待値**: スコアは変わらず連続ミスカウントが1増える
3. 「0」ボタンが他のボタンと視覚的に異なること（赤系背景）を確認する

### シナリオ 3: バースト

1. プレイヤーのスコアが 48点になるまで進める
2. 「5」ボタンをタップする（48+5=53 > 50）
3. **期待値**: バースト通知が表示され、スコアが 25点にリセットされる

### シナリオ 4: 勝利判定

1. プレイヤーのスコアが 47点になるまで進める
2. 「3」ボタンをタップする（47+3=50）
3. **期待値**: 勝利判定が働き結果画面に遷移する

### シナリオ 5: アンドゥ

1. ボタンをタップしてターンを進める
2. 「やり直し」ボタンをタップする
3. **期待値**: 直前のターンが取り消されスコアと連続ミス数が元に戻る

### シナリオ 6: 旧データクリア

1. 旧バージョン（001）のデータが localStorage に残っている状態でアプリを起動する
2. **期待値**: 旧データはクリアされ、ホーム画面が「ゲームを再開」なしの初期状態で表示される

### シナリオ 7: Mölkkout モード

1. ホーム → Mölkkout セットアップ → Mölkkout ゲーム画面を開く
2. 0〜12 のボタンが同じレイアウトで表示されることを確認する
3. 「6」をタップする
4. **期待値**: チームスコアに6が加算され次の投球者に移る

---

## ボタンサイズ確認

- iPhone 実機または Chrome DevTools モバイルエミュレータで確認する
- 1〜12 ボタンが均一サイズで、親指で無理なくタップできることを確認する
- 0 ボタンが全幅で赤系背景になっていることを確認する
