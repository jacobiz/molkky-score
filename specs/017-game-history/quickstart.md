# Quickstart: ゲーム結果履歴

**Feature**: 017-game-history
**Date**: 2026-03-06

## 統合シナリオ

### シナリオ 1: ゲーム完了 → 履歴自動保存

```
1. HomeScreen → 「新しいゲームを始める」
2. SetupScreen → プレイヤー追加 → 「ゲーム開始」
3. GameScreen → スコア入力を繰り返し → 50点プレイヤーが出現
4. ResultScreen に遷移（この時点で履歴が自動保存済み）
5. HomeScreen に戻る → 「履歴」ボタンをタップ
6. HistoryScreen に遷移 → 先ほどのゲームが一覧の先頭に表示される
```

**確認ポイント**: 手順4の終了後、localStorage の `molkky-score-history` にデータが存在する。

---

### シナリオ 2: スコアシートモーダルの確認

```
1. HistoryScreen で任意のゲームカードをタップ
2. ScoreSheetModal が開く
3. 行: ラウンド番号（全プレイヤーが1投ずつ行う1巡）、列: プレイヤー名 のテーブルを確認
4. バーストしたセル → 視覚的に強調（例: 赤背景）
5. 脱落したプレイヤー → 以降のセルがグレーアウト
6. モーダル外タップ or 「閉じる」で HistoryScreen に戻る
```

---

### シナリオ 3: 途中決着ゲームの履歴

```
1. GameScreen で「途中決着」ボタン → 確認 → ゲーム終了
2. ResultScreen → ⏱ 時間切れバッジが表示される（既存機能）
3. HistoryScreen → 当該ゲームカードに「⏱ 時間切れ」バッジが表示される
4. スコアシートモーダル → 「途中決着」の表示あり
```

---

### シナリオ 4: 履歴の削除

```
1. HistoryScreen で削除したいカードの削除ボタンをタップ
2. 確認ダイアログ表示 →「削除」
3. カードが一覧から消える
4. localStorage の `molkky-score-history` を確認 → 該当レコードが削除されている
```

---

### シナリオ 5: 最大件数超過

```
1. 通常ゲームを21件完了させる
2. HistoryScreen → 表示は最大20件
3. localStorage の `molkky-score-history.records.length` = 20
4. 最古のゲーム（21件目に追加した場合の最古）が自動削除されている
```

---

### シナリオ 6: 空状態

```
1. 履歴が0件の状態（または全削除後）で HistoryScreen を開く
2. 「まだゲーム履歴がありません」メッセージが表示される
3. ゲームカードは表示されない
```

---

### シナリオ 7: アプリ再起動後の履歴保持

```
1. ゲームを完了させて履歴に保存する
2. ブラウザタブを閉じて再度開く
3. HomeScreen → 「履歴」をタップ
4. 再起動前と同一の履歴一覧が表示される
```
