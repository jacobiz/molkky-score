# Feature Specification: ゲーム結果履歴

**Feature Branch**: `017-game-history`
**Created**: 2026-03-06
**Status**: Draft
**Input**: User description: "過去のゲーム結果履歴を残して参照したい。各履歴はターンごとのスコアシート表での表示もできるように。"

## Clarifications

### Session 2026-03-06

- Q: スコアシート詳細の表示方法（モーダル vs 別画面） → A: モーダルで表示（履歴一覧の上に重ねる）
- Q: Mölkkoutゲームの履歴スコープ → A: 今回は通常ゲームのみ対象とし、Mölkkoutは履歴保存しない（スコープ外）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 履歴一覧の参照 (Priority: P1)

ゲーム終了後、結果が自動的に保存される。ユーザーはホーム画面から「履歴」を開き、過去に行った通常ゲームの一覧（日時・参加者・勝者・ターン数）を確認できる。

**Why this priority**: 履歴機能の核心。ゲームごとに自動保存され一覧が見られるだけで、「過去の試合を振り返りたい」というニーズを満たす最小限の価値を提供する。

**Independent Test**: 通常ゲームを1件完了させ、ホーム画面の「履歴」ボタンから履歴一覧を開くと、直前のゲームが一覧に表示されていることを確認する。

**Acceptance Scenarios**:

1. **Given** 通常ゲームが終了した状態で、**When** ホーム画面の「履歴」を開くと、**Then** 直前のゲームが一覧の先頭に表示される（日時・参加者名・勝者名・ターン数を含む）。
2. **Given** 履歴が0件の状態で、**When** 「履歴」を開くと、**Then** 「まだゲーム履歴がありません」などの空状態メッセージが表示される。
3. **Given** 複数のゲームを行った状態で、**When** 「履歴」を開くと、**Then** 最新順に並んだ一覧が表示される。
4. **Given** アプリを再起動した後で、**When** 「履歴」を開くと、**Then** 再起動前と同じ履歴一覧が表示される。

---

### User Story 2 - ターンごとスコアシートの参照 (Priority: P1)

履歴一覧の任意のゲームを選択すると、スコアシートモーダルが開く。シートには各ターンにおける全プレイヤーの投点と累計スコアが表形式で並ぶ。

**Why this priority**: 「各履歴はターンごとのスコアシート表での表示もできるように」という要求の核心。履歴一覧（US1）と同等の優先度として扱う。

**Independent Test**: 保存済みのゲーム履歴を選択すると、全ターンのスコアシート表がモーダルで表示され、各プレイヤーの累計スコアの変遷を確認できることを確認する。

**Acceptance Scenarios**:

1. **Given** 履歴一覧が表示されている状態で、**When** 任意のゲームをタップすると、**Then** そのゲームのスコアシートモーダルが開く。
2. **Given** スコアシートモーダルで、**When** 画面を確認すると、**Then** 行：ラウンド番号（全プレイヤーが1投ずつ行う1巡）、列：プレイヤー名 の表形式で各ラウンドの投点と累計スコアが表示されている。
3. **Given** バーストや脱落があったゲームの詳細で、**When** スコアシートを確認すると、**Then** バースト（25点リセット）や脱落状態が視覚的に区別して表示されている。
4. **Given** 途中決着（時間切れ）ゲームの詳細で、**When** スコアシートを確認すると、**Then** 途中決着であることが明示されている。
5. **Given** スコアシートモーダルが開いている状態で、**When** モーダル外をタップするか閉じるボタンを押すと、**Then** モーダルが閉じて履歴一覧に戻る。

---

### User Story 3 - 履歴の削除 (Priority: P2)

不要な履歴を個別に削除できる。誤って保存されたゲームや不要な記録を整理するために使う。

**Why this priority**: 便利ではあるが、履歴の参照（US1・US2）が主目的であり、削除がなくても基本的なニーズは満たされる。

**Independent Test**: 履歴一覧から特定のゲームを削除すると、一覧から消えることを確認する。

**Acceptance Scenarios**:

1. **Given** 履歴一覧が表示されている状態で、**When** 特定の履歴を削除操作すると、**Then** 確認ダイアログが表示される。
2. **Given** 確認ダイアログが表示されている状態で、**When** 「削除」を選択すると、**Then** その履歴が一覧から消え、ストレージからも削除される。
3. **Given** 確認ダイアログが表示されている状態で、**When** 「キャンセル」を選択すると、**Then** 履歴は削除されず一覧が元の状態のまま残る。

---

### Edge Cases

- 履歴が最大保存件数（20件）を超えた場合、最古の履歴が自動的に削除される。
- プレイヤーが多い（最大10人）ゲームのスコアシートは横スクロールで対応する。
- スコアシートのデータが破損・不整合な場合は「詳細を表示できません」と表示し、一覧からは削除できる。
- Mölkkoutゲームは本フィーチャーの対象外とする（履歴は保存しない）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 通常ゲームが終了したとき、システムは結果をゲーム履歴として自動保存しなければならない。
- **FR-002**: システムはホーム画面から履歴一覧へアクセスするエントリーポイントを提供しなければならない。
- **FR-003**: 履歴一覧には各ゲームの終了日時・参加者名・勝者・ターン数を表示しなければならない（引き分けの場合は勝者の代わりに引き分けであることを示す）。
- **FR-004**: ユーザーは履歴一覧から任意のゲームをタップしてスコアシートモーダルを開くことができなければならない。
- **FR-005**: スコアシートモーダルはラウンド番号・各プレイヤーの投点・累計スコアを表形式で表示しなければならない（1行＝全プレイヤーが1投ずつ行う1ラウンド）。
- **FR-006**: バースト・脱落・途中決着（時間切れ）はスコアシート上で視覚的に識別できなければならない。
- **FR-007**: 履歴はデバイスにローカル保存され、アプリ再起動後も参照できなければならない。
- **FR-008**: ユーザーは任意の履歴を個別に削除できなければならない（削除前に確認ダイアログを表示する）。
- **FR-009**: システムは最大20件の履歴を保持し、超過した場合は最古のものを自動削除しなければならない。
- **FR-010**: 履歴が0件の場合、空状態メッセージを表示しなければならない。

### Key Entities

- **GameHistoryRecord**: 通常ゲーム1件の記録。属性：ID・終了日時・参加者一覧・勝者・総ターン数・途中決着フラグ・ターンごとの投球記録。
- **TurnEntry**: 1投球の記録。属性：通算投球番号・プレイヤー名・投点・累計スコア・バーストフラグ・脱落フラグ。スコアシート表示時はプレイヤーごとの投球回数でラウンドに集約して表示する。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 通常ゲーム終了後、ユーザーが追加操作をしなくても結果が自動保存され、履歴一覧から即座に参照できる。
- **SC-002**: 20件の履歴を保存した状態でも、一覧表示が1秒以内に完了する。
- **SC-003**: スコアシートモーダルで、全ターンの全プレイヤーのスコアを一つの画面（スクロール可）で確認できる。
- **SC-004**: アプリを完全に終了して再起動した後でも、履歴データが失われない。
- **SC-005**: 日本語・英語・フィンランド語の全言語でUIが正しく表示される。

## Assumptions

- 履歴の最大保存件数は20件とする（デバイスのストレージ容量制限を考慮した妥当な上限）。
- 履歴の全削除（一括クリア）機能はスコープ外とする。
- ゲーム中に途中終了（ホーム画面へ戻る）した場合は履歴として保存しない。
- Mölkkoutゲームの履歴は本フィーチャーのスコープ外とし、将来の別フィーチャーとして対応する。
