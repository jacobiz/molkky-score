# Feature Specification: 次のゲーム設定画面（リマッチ）

**Feature Branch**: `009-rematch-setup`
**Created**: 2026-03-01
**Status**: Draft
**Input**: User description: "ゲーム終了後、次のゲームに進む際にプレイヤーの追加・削除や投球順変更ができるようにしたい"

## Clarifications

### Session 2026-03-01

- Q: 結果画面の「もう一度」ボタンをどう扱うか？ → A: ボタンラベルは「もう一度」のまま変えず、動作を「前回プレイヤー引き継ぎのセットアップ画面へ遷移」に変更する

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 前回のプレイヤーを引き継いで次のゲームを準備する (Priority: P1)

ゲームが終了して結果画面が表示された後、「もう一度」ボタンを押すと、前回ゲームの
プレイヤー一覧が初期値として設定済みのセットアップ画面が開く。ここでプレイヤーの追加・削除・
並び替え・シャッフルを行ってから「ゲーム開始」を押すと、変更後の構成で次のゲームが始まる。
変更しない場合はそのまま「ゲーム開始」を押せば即座にリスタートと同等の動作になる。

**Why this priority**: ゲームを連続してプレイする際の最も一般的な操作。毎回全プレイヤーを
入力し直す手間がなく、かつ新しいプレイヤーの追加や投球順の変更ができることが、
リピートプレイ体験の核心。

**Independent Test**: ゲームを完了し結果画面で「もう一度」ボタンを押したとき、
前回ゲームのプレイヤー一覧が表示されているセットアップ画面が開くことを目視確認する。
プレイヤーを1名削除し別の名前を追加してから「ゲーム開始」を押すと、変更後のメンバーで
ゲームが始まることを確認する。

**Acceptance Scenarios**:

1. **Given** ゲームが終了して結果画面が表示されている、**When** 「もう一度」ボタンを押す、**Then** 前回ゲームのプレイヤー一覧（名前・順序）が初期値として設定済みのセットアップ画面が表示される
2. **Given** 前回プレイヤーが設定済みのセットアップ画面、**When** プレイヤーを追加・削除・並び替えして「ゲーム開始」を押す、**Then** 変更後のプレイヤー構成で新しいゲームが開始される
3. **Given** 前回プレイヤーが設定済みのセットアップ画面、**When** 何も変更せず「ゲーム開始」を押す、**Then** 前回と同じプレイヤー構成でゲームが開始される
4. **Given** 前回プレイヤーが設定済みのセットアップ画面、**When** 「シャッフル」ボタンを押す、**Then** プレイヤーの順序がランダムに並び替えられる
5. **Given** 結果画面、**When** 「新しいゲーム」ボタンを押す、**Then** 空のセットアップ画面が開く（既存動作を維持）

---

### Edge Cases

- 前回ゲームで脱落（eliminated）したプレイヤーも次のゲームの初期リストに含まれる（スコアはリセット）
- 前回のプレイヤーが10名（上限）の場合、セットアップ画面では追加ボタンが無効になる
- 前回のプレイヤーが1名の場合（通常は発生しないが）、「ゲーム開始」は2名未満エラーになる
- プレイヤーを全員削除した状態で「ゲーム開始」を押すと、既存の最小人数エラーが表示される
- リマッチ設定中にブラウザをリロードした場合、通常のセットアップ画面として機能する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 結果画面の「もう一度」ボタンの動作を、前回ゲームのプレイヤー一覧（名前・投球順）が初期値として設定済みのセットアップ画面への遷移に変更すること（ボタンラベルは「もう一度」のまま維持）
- **FR-002**: リマッチ用セットアップ画面で、プレイヤーの追加・削除・並び替え（↑↓）・シャッフルができること
- **FR-003**: リマッチ用セットアップ画面から「ゲーム開始」を押すと、その時点のプレイヤー構成でゲームが開始されること
- **FR-004**: 「新しいゲーム」ボタンは空のセットアップ画面へ遷移する既存動作を維持すること
- **FR-005**: リマッチ用セットアップ画面における最小2名・最大10名のバリデーションが機能すること

### Key Entities

- **RematchPlayers**: 前回ゲームから引き継がれるプレイヤー名の順序付きリスト（スコアなし、名前のみ）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 結果画面で「もう一度」ボタンを押すと、前回プレイヤー全員が設定済みのセットアップ画面が表示される（目視確認）
- **SC-002**: リマッチ設定画面でプレイヤーを追加・削除・並び替えして開始したゲームに、変更が反映されている（目視確認）
- **SC-003**: 変更なしで「ゲーム開始」を押した場合、前回と同じメンバー・順序でゲームが始まる（目視確認）
- **SC-004**: 「新しいゲーム」ボタンは引き続き空のセットアップ画面へ遷移する（目視確認）
- **SC-005**: 既存の全ユニットテストが引き続きパスすること

## Assumptions

- 「もう一度」ボタンのラベルは変更しない。動作のみ変更する（即時リスタート → pre-populated SetupScreenへ遷移）
- 前回ゲームで脱落したプレイヤーも次ゲームに含める（除外するかどうかの設定は本機能のスコープ外）
- プレイヤー名は変更不可（名前変更は本機能のスコープ外）
- Mölkkout 画面への遷移ボタンは本機能の対象外（現状維持）
