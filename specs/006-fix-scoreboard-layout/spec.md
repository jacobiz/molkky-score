# Feature Specification: Fix Scoreboard Layout for 5-6 Players

**Feature Branch**: `006-fix-scoreboard-layout`
**Created**: 2026-03-01
**Status**: Draft
**Input**: User description: "プレイヤー数が5,6人のときに、プレイヤー一覧の下が得点入力に隠れてしまっている(スライドバーが有効になっている)ため、常に全体が表示されるようにレイアウトを整えたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 全プレイヤーを常に一覧表示する (Priority: P1)

5〜6人でゲームを行うとき、プレイヤー一覧と得点入力エリアが同時に画面内に収まり、プレイヤー一覧内でスクロールしなくても全員のスコアが確認できる。得点を入力しながら全員の状況を把握できるため、ゲーム進行がスムーズになる。

**Why this priority**: 5〜6人は標準的なプレイ人数であり、現状のレイアウト崩れはゲームプレイの妨げになる致命的なバグ相当のUX問題。

**Independent Test**: 6人でゲームを開始し、プレイヤー一覧内にスクロールバーが出ないこと、全員のスコアが見えること、得点入力エリアも同時に画面内に表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** 6人でゲームが進行中、**When** ゲーム画面を開く、**Then** プレイヤー一覧内でスクロールせずに全6人が表示され、得点入力エリアも同じ画面内に見える
2. **Given** 5人でゲームが進行中、**When** ゲーム画面を開く、**Then** プレイヤー一覧内でスクロールせずに全5人が表示され、得点入力エリアも同じ画面内に見える
3. **Given** 2〜4人でゲームが進行中、**When** ゲーム画面を開く、**Then** 従来通りすべてのプレイヤーと得点入力エリアが問題なく表示される
4. **Given** 6人でゲームが進行中、**When** 脱落者が出てプレイヤー数が減る、**Then** 引き続き全員が一覧に表示される

---

### User Story 2 - 得点ボタンの文字を大きくする (Priority: P2)

得点入力エリアの各ボタン（0〜12点）に表示される数字が現在より大きく表示され、ゲーム中の素早い操作でも正確にタップできる。屋外での使用や手袋着用時など、視認性が低い状況でも安心して操作できる。

**Why this priority**: 視認性向上はゲーム体験を直接改善するが、US1（レイアウト崩れ修正）ほど機能的なブロッカーではないため P2。

**Independent Test**: ゲーム画面の得点入力エリアを開き、数字ボタン（0〜12）の文字が以前より明らかに大きく表示されることを目視確認する。

**Acceptance Scenarios**:

1. **Given** ゲームが進行中、**When** 得点入力エリアを確認、**Then** 0〜12の各ボタンの数字が以前（v0.1.0）より大きく表示されている
2. **Given** 得点ボタンの文字サイズを変更した後、**When** ボタンをタップして得点を入力、**Then** 得点入力の動作に変化はなく正常に機能する

---

### Edge Cases

- 最大人数（6人）全員が脱落せず残っている状態でもレイアウトが崩れないこと
- デバイスの画面高さが極端に小さい場合（320px 未満など）は許容範囲外とする
- 横向き（landscape）表示は既存の md: レイアウトが対応しているため対象外

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: プレイヤー数が2〜6人のいずれの場合も、ゲーム画面でプレイヤー一覧全体と得点入力エリアが同時に1画面内に収まること
- **FR-002**: プレイヤー一覧の中でスクロール操作が不要であること（一覧内スクロールバーが表示されないこと）
- **FR-003**: 得点入力エリアが画面外に追いやられたり、プレイヤー一覧に隠れたりしないこと
- **FR-004**: 既存のゲームロジック（スコア計算・undo・ターン管理）に影響を与えないこと
- **FR-005**: 横向き表示（タブレット・デスクトップ）の既存レイアウトを維持すること

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 6人ゲームにおいて、ゲーム画面のプレイヤー一覧内にスクロールバーが表示されない
- **SC-002**: 6人ゲームにおいて、得点入力エリアが画面内に完全に表示される
- **SC-003**: 2〜4人ゲームにおいて、機能的な動作に変化がないこと（スコア表示・入力・undo が正常に動作する）。レイアウト最適化に伴う行高さ等の軽微な視覚変化は許容する。
- **SC-004**: 既存の全ユニットテスト（27件）が引き続きパスする

## Assumptions

- 対象はモバイル縦向き（portrait）表示のみ。横向き（md: ブレークポイント以上）は既存レイアウトを維持
- 最低対応画面高さは 600px（一般的なスマートフォンの最小高さ）
- プレイヤー行の高さは均等であり、6人分が1画面に収まるよう両エリアの高さを動的に分配する
- データモデル・ゲームロジックへの変更は一切不要
