# Feature Specification: スコアシート印刷機能

**Feature Branch**: `018-scoresheet-print`
**Created**: 2026-03-06
**Status**: Draft
**Input**: User description: "スコアシート印刷機能を追加したい"

## Clarifications

### Session 2026-03-06

- Q: 結果画面の印刷ボタンを押した時のフローは？モーダル表示を経由するか直接印刷か → A: モーダルを先に開き、内容確認後にモーダル内の印刷ボタンで印刷する（US1 と同一 UX）

## User Scenarios & Testing

### User Story 1 - 履歴スコアシートの印刷 (Priority: P1)

大会の審判や記録係が、ゲーム終了後に履歴画面のスコアシートを印刷して公式記録として残したい。

**Why this priority**: 大会利用の主要ニーズ。スコアシートモーダルは既に実装済みであり、印刷レイアウトを追加することで即座に公式記録用途に使えるようになる。

**Independent Test**: 履歴画面のスコアシートモーダルを開き、印刷ボタンを押してブラウザの印刷ダイアログが表示されることを確認。印刷プレビューにはスコアシートのみが表示され、ナビゲーションボタン類は表示されない。

**Acceptance Scenarios**:

1. **Given** 履歴画面でスコアシートモーダルが開いている状態で、**When** 「印刷」ボタンを押すと、**Then** ブラウザの印刷ダイアログが開く
2. **Given** 印刷ダイアログが開いている状態で、**When** 印刷プレビューを確認すると、**Then** スコアシートテーブル・試合日時・参加者・勝者のみが印刷領域に表示され、「閉じる」「印刷」ボタン等のUI要素は表示されない
3. **Given** 印刷プレビューを確認すると、**When** 印刷レイアウトを見ると、**Then** 全ラウンドのスコアが切れずに表示される

---

### User Story 2 - 結果画面からのスコアシート印刷 (Priority: P2)

ゲーム終了直後の結果画面から、そのゲームのスコアシートをモーダルで確認してから印刷したい。履歴を遡る手間なくその場で記録を残せる。

**Why this priority**: 大会の現場では、ゲーム終了直後に記録用紙を渡す運用が多い。結果画面からワンタップでスコアシートを開き印刷できると実用性が高まる。

**Independent Test**: 通常ゲームを完了して結果画面を表示し、「スコアシートを印刷」ボタンからスコアシートモーダルが開き、そのモーダルの印刷ボタンで印刷ダイアログを開けることを確認。

**Acceptance Scenarios**:

1. **Given** ゲームが終了して結果画面が表示されている状態で、**When** 「スコアシートを印刷」ボタンを押すと、**Then** そのゲームのスコアシートモーダルが開く
2. **Given** 結果画面のスコアシートモーダルが開いている状態で、**When** モーダル内の印刷ボタンを押すと、**Then** ブラウザの印刷ダイアログが起動する
3. **Given** 印刷プレビューを確認すると、**When** 内容を見ると、**Then** 試合日時・参加者全員の名前・ラウンドごとのスコア・勝者が正確に表示されている

---

### Edge Cases

- プレイヤー数が多い（最大10名）場合、テーブルの横幅が用紙をはみ出す可能性がある。横方向スクロールではなく用紙に収まるよう自動縮小して表示する
- ラウンド数が多い場合（例：50ラウンド以上）、縦方向に複数ページにまたがる。各ページにテーブルヘッダー（プレイヤー名の列）を繰り返し表示する
- 印刷ボタンはスクリーン表示時のみ有効とし、印刷用レイアウトでは非表示にする

## Requirements

### Functional Requirements

- **FR-001**: スコアシートモーダルに「印刷」ボタンを追加し、押下時にブラウザの印刷ダイアログを開く
- **FR-002**: 印刷レイアウトでは操作ボタン類（閉じるボタン、印刷ボタン）・モーダル外の背景を非表示にし、スコアシートのみを表示する
- **FR-003**: 印刷ヘッダーには試合日時・参加者（投球順）・勝者名・終了理由を表示する
- **FR-004**: プレイヤー数が多くテーブルが横幅をはみ出す場合、印刷時は自動縮小して用紙内に収める
- **FR-005**: ラウンド数が多く複数ページになる場合、各ページのテーブルヘッダー（プレイヤー名行）を繰り返し表示する
- **FR-006**: 結果画面に「スコアシートを印刷」ボタンを追加し、押下するとスコアシートモーダルが開く。ユーザーはモーダル内容を確認後、モーダルの印刷ボタンで印刷ダイアログを起動する（US1 と同一の印刷フロー）

### Key Entities

- **スコアシート（GameHistoryRecord）**: 既存エンティティ。試合日時・参加者・勝者・全ターンの投球記録を含む。印刷機能はこのデータをそのまま使用する

## Success Criteria

### Measurable Outcomes

- **SC-001**: 印刷ボタン押下から印刷ダイアログ表示まで1秒以内に完了する
- **SC-002**: 印刷プレビューにスコアシート以外のUI要素（ボタン、背景）が表示されない
- **SC-003**: 2〜10名のプレイヤー・任意のラウンド数のスコアシートが用紙サイズ内に収まって印刷される
- **SC-004**: 印刷後の画面表示は印刷前と同一の状態に戻る（モーダルが閉じたり状態が壊れない）

## Assumptions

- 印刷機能はブラウザ標準の印刷機能を利用する。PDF生成ライブラリ等の追加は行わない
- 用紙サイズはA4横向きを基準とするが、ブラウザの用紙設定に依存するため、アプリ側でサイズを固定はしない
- 結果画面からの印刷（US2）は、ゲーム履歴に保存済みのデータを使用する。現状の実装ではゲーム終了時に自動保存される
