# Molkky Score Constitution

## Core Principles

### I. シンプルさ優先

機能はシンプルであることを最優先とする。YAGNI を徹底し、現在の要件に必要最低限の
実装のみを行う。複雑さを導入する場合は明確な正当理由が必要である。

- コンポーネントは単一の責務を持つこと（MUST）
- 追加ライブラリ・依存関係は最小限に抑えること（MUST）
- 将来の拡張を先取りした設計は行わないこと（MUST NOT）
- 既存の実装で解決できる場合、新しい抽象化を作らないこと（MUST）

**根拠**: スコアアプリはゲーム中に素早く操作するものであり、シンプルな実装はバグを
減らしメンテナンスを容易にする。

### II. テストファースト（主要機能のみ）

主要機能（スコア計算・ゲームルール）に対してはテスト駆動開発（TDD）を適用する。
テストは実装前に記述し、まず失敗することを確認してから実装を開始する。
補助的な UI 要素や単純なユーティリティへのテストは不要。

- 主要機能のテストは実装前に記述すること（MUST）
- テストは必ず失敗状態を確認してから実装を開始すること（MUST）
- テスト対象は主要機能（スコア計算・ゲームルール）のみとすること（MUST）
- UI コンポーネントや単純なユーティリティのテストは任意（MAY）

**根拠**: 主要機能のバグはゲーム体験を損なう。一方で全コードへのテストはオーバー
エンジニアリングであり、シンプルさ優先の原則と矛盾する。

### III. モバイルファースト・PWA

このアプリはモバイルデバイスでゲーム中に使用されることを想定する。オフラインでも
動作し、スマートフォンで快適に操作できることを保証する。

- UI はモバイル（スマートフォン）を最優先で設計すること（MUST）
- オフライン動作を保証すること（MUST）
- タッチ操作に最適化すること（MUST）
- 主要操作のレスポンスタイムは 100ms 以内を目標とすること（SHOULD）

**根拠**: Mölkky は屋外のゲームであり、プレイヤーはモバイルデバイスを使用する。
Wi-Fi が利用できない環境でも動作することが重要。

## 開発標準

技術スタックと開発規約を定義する。

- **言語**: TypeScript（strict mode 有効）
- **フレームワーク**: React 19
- **ビルドツール**: Vite
- **PWA**: vite-plugin-pwa
- **UI 言語**: 日本語（ユーザー向け文字列はすべて日本語）
- **コード**: 変数名・コメントは英語を基本とする

## 品質ゲート

各機能のリリース前に以下を確認する。

- 主要機能のテストがすべてパスしていること（MUST）
- モバイル（iOS / Android）での動作確認（MUST）
- オフライン動作確認（MUST）
- TypeScript のコンパイルエラーがないこと（MUST）
- シンプルさ原則に違反する不必要な複雑さがないこと（MUST）

## Governance

本コンスティテューションはプロジェクトの全実践に優先する。
改定には変更内容の文書化とバージョン更新が必要である。

- すべての PR・レビューは本原則への適合を検証すること（MUST）
- 複雑さの導入は plan.md の「Complexity Tracking」表で正当化されなければならない（MUST）
- 改定はセマンティックバージョニングに従う:
  - MAJOR: 原則の廃止・根本的な再定義
  - MINOR: 新原則の追加・大幅な拡張
  - PATCH: 文言の明確化・誤字修正
- 開発ガイダンスについては本コンスティテューションを参照すること
